# Empathize Question Bank

Categorized open questions for the Empathize phase (Step 1, Phase A). Use these conversationally — pick the most natural follow-up based on what the user just said. Never run through them as a checklist.

## Opening Questions

These start the conversation. Pick ONE based on context:

- "能告诉我你们目前面临的业务挑战是什么吗？是什么促使你想做这个产品/服务？"
- "聊聊你们团队/客户现在的工作方式？什么事情促使你觉得需要一个新产品？"
- "你们的用户现在每天是怎么工作的？能描述一下他们的日常吗？"

## Understanding Daily Reality

Used when the user mentions a role but hasn't described their day-to-day:

- "你们的 [角色] 一天的工作是什么样的？"
- "他们最头疼的事情是什么？"
- "现在他们是怎么解决这些问题的？用了哪些工具或系统？"
- "他们一天里花最多时间在做什么？"
- "他们的工作流程里，哪些步骤最容易出错或返工？"

## Surfacing Hidden Pain Points

Used when answers are surface-level ("挺好的"、"还行"):

- "有没有什么操作特别繁琐，但大家已经习以为常了？"
- "有没有哪些事情是他们想做但做不到的？"
- "他们的上级/客户对他们有什么期望是目前满足不了的？"
- "如果有一个魔法按钮能帮他们做一件事，他们最希望是什么？"
- "最近有没有出过什么问题让大家很头疼？能具体说说吗？"

## Understanding Current Solutions

Used when you need to understand the "as-is" state:

- "他们现在用什么系统/工具来做这件事？"
- "现有方案最大的不足是什么？"
- "他们有没有自己想过变通的办法？比如用 Excel、微信群之类的？"
- "现有系统是谁提供的？用了多久了？"
- "如果要把现有系统换掉，最大的顾虑是什么？"

## Understanding Stakeholder Ecosystem

Used when multiple roles or organizations are involved:

- "除了 [已提到的角色]，还有谁会用到这个产品？或者谁会间接受影响？"
- "不同角色之间是怎么协作的？信息怎么传递？"
- "谁是最终决策者？谁是日常使用者？他们的诉求一样吗？"
- "客户那边有没有 IT 团队？他们的角色是什么？"

## Understanding Success Criteria

Used when you need to anchor "done" / "good":

- "如果这个产品做成了，你怎么知道它成功了？"
- "你最在意的一个指标是什么？"
- "用户会怎么评价这个产品好不好用？"
- "你的领导/客户会怎么衡量这个项目的成功？"

## Understanding Constraints

Used to surface hidden boundaries:

- "有没有什么技术限制我们必须考虑的？比如现有系统、网络环境、设备类型？"
- "时间上有什么预期？有没有必须赶上的节点？"
- "预算方面有什么约束？"
- "有没有合规或安全方面的硬性要求？"

## Deepening Vague Statements

Used when the user gives abstract/vague answers:

| User says | Follow-up |
|---|---|
| "很复杂" | "复杂在哪？是流程复杂、文档不清楚、还是概念本身难理解？" |
| "做不到" | "是不知道能做、知道但做不了、还是做了但效果不好？" |
| "数据不够" | "具体缺什么数据？是覆盖不全、粒度不够、还是时效性不行？" |
| "效率低" | "哪个环节效率低？大概要花多少时间？理想情况应该是多久？" |
| "不好用" | "哪里不好用？是找不到功能、操作步骤多、还是结果不准确？" |
| "有需求" | "需求是谁提的？在什么情况下产生的？他们现在怎么应对？" |

## Tips for the Agent

1. **Don't exhaust all questions** — stop when you have enough to move to Phase B
2. **Follow the thread** — when the user opens a topic, dig deeper there before switching
3. **Use their words** — repeat back their terminology, don't replace with formal terms prematurely
4. **Notice what's unsaid** — if they talk about one role extensively but barely mention another, there's a reason
5. **Emotion signals** — frustration, enthusiasm, resignation all carry information about priority
